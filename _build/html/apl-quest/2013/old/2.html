Making the Grade#

Problem: Write a dfn which returns the percent (from 0 to 100) of passing (65 or higher) grades in a vector of grades.

Video: https://youtu.be/pxo2BtoMxP4 Code: abrudz/apl_quest

Example Solutions: 2 Scaler functions and a reduction over a vectorized operation.

F ← {100×(+/⍵≥65)÷≢⍵}
J ← 100×+.≥∘65÷≢ ⍝ Tacit

Explanation:

F ← {100×(+/⍵≥65)÷≢⍵}

  1. (+/⍵≥65) use greater than or equal to as a Comparison Function returning a boolean vector of 1 if true and 0 if false.

  2. Then sum the result - +/.

  3. ÷≢⍵ Tally the length of your argument and divde that by the result in step 1.

  4. Multiply this by 100 to get your percentage of passing grades.

Performance:

F ← {100×(+/⍵≥65)÷≢⍵} ⍝ n:≥ n-1:+ 1:÷ 1:×
G ← {100×+/(⍵≥65)÷≢⍵} ⍝ n:≥ n-1:+ n:÷ 1:×
H ← {+/100×(⍵≥65)÷≢⍵} ⍝ n:≥ n-1:+ n:÷ n:×

Note that it does not matter where we do the sum mathmatically. However there are siginificant performance penalties for summing after the division.

s←?1e6⍴100
'cmpx'⎕cy'dfns'
cmpx'F s' 'G s' 'H s'
⍝ F s → 6.6E¯5 | 0%
⍝ G s → 2.4E¯3 | +3451%
⍝ H s → 2.8E¯3 | +4126%

We can evalute the performance of each function by importing the CMPX function from the DFNS library.

  1. We Roll1 million 1e6 random numbers between 1 and 100

  2. We copy ⎕cy CMPX from the DFNS library into our workspace

  3. We use cmpx to evalute the performance of each function. With the first function as our baseline.

F ← {100×(+/⍵≥65)÷≢⍵}
I ← {100×(⍵+.≥65)÷≢⍵}
J ← 100×+.≥∘65÷≢ ⍝ Tacit
K ← 100×+.≤÷≢⍤⊢

We are dealing with a scaler (65) and a vector (Scores). We should notice this pattern. We have a sum over a comparison of vectors. When we have that, we should think, Inner Product.

  1. +.≥ Replace reduce in +/ with inner product

  2. (J) 65 is a Bound constant to the inner product allowing us to remove the parenthesis.

  3. J is parsed as {100×((⍵(+.≥)65)÷(≢⍵))}

Generalization: Adding the cutoff point as an additional argument

  1. K is parsed as {100×((⍺(+.≤)⍵)÷(≢⍵))}

  2. We are reversing the ≥ so that the Cuttoff Point (65) can be taken as the left argument. Represented by ⍺ in step 1.

  3. ≢⍤⊢ We use Atop so that we can apply Tally monadically. (Dyadic Tally is Not Match) We could also use Jot ≢∘⊢ in this case and achieve the same result.
